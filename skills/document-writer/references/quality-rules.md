# 文档质量检查规则

生成文档的最后一步，逐条检查以下规则。不通过 → 修正 → 重新检查 → 直到全部通过。

## 禁止模式（Q-01 到 Q-06）

### Q-01：占位符检测

**检测关键词**（不区分大小写，含变体）：

```
TODO, FIXME, TBD, WIP, Coming soon, COMING SOON,
待办, 待完成, 待补充, 占位, 占位符, 后续补充,
Work in progress, UNDER CONSTRUCTION, tbd, wip
```

**例外**：若在 CONTRIBUTING.md 中引用项目的公开 Roadmap，且链接到 Issue/TAPD，允许出现。

**违规处理**：删除该占位符及其所在的句子。若整个章节因此变空，删除该章节。

### Q-02：无输出的命令

**检测模式**：

```
# 违规：命令块之后无输出
```bash
$ npm install
```

# 合规：命令块之后有输出
```bash
$ npm install
# → added 142 packages in 3s
```
```

**适用场景**：安装命令、CLI 命令、脚本执行、API 调用结果。

**违规处理**：补充输出或效果描述。若无法确定真实输出，标注 `<!-- 输出需验证 -->`。

### Q-03：无输入输出的 API 描述

**检测模式**：

```
# 违规：只描述功能，无示例
### `fetch(url)`
发送 HTTP 请求。

# 合规：完整的输入输出
### `fetch(url, options?)`
发送 HTTP 请求。

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| url | string | 是 | 请求地址 |
| options | object | 否 | 请求配置 |

**返回值**：`Promise<Response>`

**示例**：
```js
const res = await client.fetch("/api/users").json()
// → { data: [{ id: 1, name: "Alice" }], total: 1 }
```
```

**违规处理**：补充黄金模板格式。若信息不足，标注 `<!-- 参数从代码推断，请验证 -->`。

### Q-04：推卸式引用

**检测模式**：

```
See source code for details
请查看源码
源码里有注释
refer to the source
详见代码实现
详见源代码
```

**违规处理**：改为内联概要说明。若确实无法概括，改为：

> 实现细节见 `{文件路径}:{行号}` 的 `{函数名}` 函数。

### Q-05：模糊表述

**检测模式**：

```
Supports all options
支持所有选项
Various options available
all standard features
多种方式
各种配置
```

**违规处理**：列出具体选项/配置。若数量太多（> 20 个），列出最常用的 5 个 + 链接到完整列表。

### Q-06：空心章节

**检测方式**：章节标题后有内容但内容 < 20 个词，且不包含代码块或列表。

**违规处理**：补足内容，或删除该章节并从导航中移除链接。

## 结构完整性检查

### README 检查

- [ ] 概述章节包含三要素（做什么 + 解决什么 + 与方案对比）
- [ ] 安装章节包含前置条件 + 安装命令 + 验证方法
- [ ] 快速开始章节可在 30 秒内读完
- [ ] 每个子文档导航链接附有一句话说明
- [ ] README 总行数 ≤ 120 行

### 子文档检查

- [ ] 概述章节说明该文档与 README 的关系
- [ ] 底部有「相关文档」章节，链接回 README 和其他子文档
- [ ] 独立可读（不依赖 README 上下文即可理解）

---

## 示例完整性检查

### API/接口检查

对每个对外暴露的接口：

- [ ] 有完整的参数说明（名称 + 类型 + 是否必填 + 说明）
- [ ] 有返回值类型和说明
- [ ] 有可执行的示例代码
- [ ] 有示例输出（实际输出或预期输出描述）
- [ ] 有边界情况或注意事项说明（若有的话）

### CLI 命令检查

对每个命令：

- [ ] 有完整的命令格式（含所有选项）
- [ ] 有终端输出示例（粘贴或合理构造）
- [ ] 选项有类型和默认值说明

### 配置项检查

对配置文档中的每个配置项：

- [ ] 有类型说明
- [ ] 有默认值
- [ ] 有使用场景或效果说明

---

## 内容一致性检查

- [ ] 子文档间无重复内容（同一信息只在一处写全，其他处链接引用）
- [ ] 所有内部链接（指向其他 .md 文件）的文件名与实际生成的对应
- [ ] README 中的 API/命令速览与子文档中的完整描述不矛盾
