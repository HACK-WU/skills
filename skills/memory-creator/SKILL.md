# 记忆内容生成器

## 功能说明

本 skill 指导 AI 生成简洁的记忆内容描述，**仅做输出，不写入记忆系统**。生成的记忆内容供用户参考或手动保存。

## 生成规则

### 核心原则

- **一句话**：每个记忆内容必须精简为一句话
- **简洁凝练**：去除冗余词汇，保留核心信息
- **独立完整**：单句即可理解完整含义

### 格式要求

```
记忆：{主语} + {动作/状态} + {关键信息}
```

### 生成流程

1. **提取核心信息**：从用户输入中识别关键事实
2. **去除冗余**：删除修饰词、重复信息、上下文废话
3. **重组语句**：按"主语+动作+信息"结构重组
4. **输出记忆**：以"记忆："前缀输出

## 示例

### 示例 1：用户描述较长

**用户输入**：
"我今天在调试那个登录功能的时候发现，原来是因为JWT token过期时间设置得太短了，只有15分钟，用户经常在操作过程中就过期了，需要改成24小时"

**输出**：
```
记忆：JWT token 过期时间需从 15 分钟改为 24 小时
```

### 示例 2：用户描述简短

**用户输入**：
"数据库密码是 abc123"

**输出**：
```
记忆：数据库密码为 abc123
```

### 示例 3：用户描述模糊

**用户输入**：
"那个API好像有问题，之前用的时候返回的数据格式不对，好像是字段名变了"

**输出**：
```
记忆：某 API 返回数据格式异常，疑似字段名变更
```

## 注意事项

1. **不自动保存**：生成的记忆内容仅输出，不调用 memory_store
2. **不重复生成**：同一内容不重复生成记忆
3. **敏感信息**：涉及密码、密钥等敏感信息时，提醒用户注意安全
4. **模糊信息**：用户描述模糊时，保留不确定性表述（如"疑似"、"可能"）

## 跳过条件

以下情况不生成记忆：
- 用户明确说"不用记住"、"不需要记忆"
- 内容过于简单（如"好的"、"知道了"）
- 内容为问题而非陈述（如"这个怎么用？"）
