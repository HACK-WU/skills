# 对比例子索引

以下示例展示「AI 默认输出风格」与「本 skill 输出风格」的差异。

| 示例 | 项目类型 | 说明 |
|------|----------|------|
| [example-1-library.md](./example-1-library.md) | 库/SDK | 一个 HTTP 请求库的 README |
| [example-2-cli.md](./example-2-cli.md) | CLI 工具 | 一个文件批量处理 CLI 工具 |

每个示例中，「Before」代表典型 AI 默认生成的文档，「After」代表本 skill 策略下的产出。

核心差异主要体现在：
1. 是否有完整的输入输出示例
2. 项目类型是否影响文档结构
3. 是否有空洞占位符和模糊表述
4. README 长度控制和导航设计
